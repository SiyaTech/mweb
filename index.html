Engineered for Security, Built for Autonomy

Forget manual logging tools. Munshiji runs seamlessly, protects your data with robust localized models, and provides powerful integration targets.

Self-Healing Sync Worker

Accidentally delete a spreadsheet row? No problem. Munshiji scans remote Google Sheets, compares histories with local SQLite data, and automatically re-inserts any missing records.

Duplicate Protection (3-Min Window)

Carriers or banks sometimes broadcast redundant SMS alerts. Munshiji performs fuzzy timestamp-bracket matching on account, bank, and transaction details to guarantee clean records.

Real-Time Webhook Target

For power users and businesses. Register a custom URL in settings; Munshiji immediately dispatches parsed JSON transactions as HTTP POST requests for continuous backend processing.

Encrypted SQL Storage

All SMS message logs can be fully encrypted before storing in the local Room database. Munshiji never logs, sends, or stores your personal messages or credit card details on third-party clouds.

POS Tenant Secure Tunnel

Link multiple local billing counters or POS terminals to an Admin device. Munshiji broadcasts transaction logs instantly across secure peer-to-peer tunnels to synchronize central store records.

Direct Google Integration

Uses official Google API integration tokens securely resolved inside Android Play Services. Your Google Account password is never entered, requested, or accessed by the app.

How Munshiji Simplifies Your Bookkeeping

Four simple steps, completed in milliseconds, entirely behind the scenes without user intervention.

1

Carrier SMS Received

Your Android device receives an institutional SMS from your bank (e.g. AD-HDFCBK) regarding a transaction.

2

On-Device Extraction

Munshiji's core parser extracts the exact amount, transaction type (Cr/Dr), cards/account numbers, and reference code.

3

Google Sheet Row Append

The background SyncWorker appends the record into your Google Sheet. Works offline and syncs immediately when online.

4

Real-time Dashboard

Log in on your sheet or check the app dashboard charts. Review instant insights, share sheets, or analyze monthly expenditure.

Power Integration for Developers & Admins

Munshiji is built with extensible interfaces. Whether you want to dump data into your web application, sync with automation tools, or run a POS multi-terminal store, we have got you covered.

HTTP POST JSON Payload Format content-type: application/json

Whenever a transaction SMS is parsed on the Admin phone, it instantly triggers a background HTTP POST with this standardized JSON structure: 

{
  "id": "tx_1720726712390",
  "sender": "AD-HDFCBK",
  "timestamp": 1720726712000,
  "amount": 1250.00,
  "txnType": "DEBIT",
  "medium": "UPI",
  "account": "A/c XX5432",
  "referenceNo": "419283719283",
  "category": "Dining",
  "rawBody": "Your a/c ending XX5432 has been debited by Rs 1250..."
}

* In Multi-Terminal POS mode, clients connected via Secure local NS-SD Tunnels will also receive this live broadcast.

Quick Start & Setup Guide

Configure Munshiji in less than two minutes with our step-by-step instructions.

📥

1. Install and Permissions

Download the compiled app-debug.apk to your Android device. Run the installer and grant the SMS Receive and Read permissions when prompted. This is critical for the background ledger engine to automate logging.

System Requirement: Android 8.0+
🔗

2. Link Google Spreadsheet

Open Google Sheets on a browser and create a new sheet. Copy the long string of characters from the URL:
/d/<Spreadsheet_ID>/edit.
Paste this ID into the Spreadsheet ID field under the app Settings tab.

Direct Official Google OAuth Flow
✨

3. Verify and Run

Tap the "Run Diagnostics" button in settings to ensure database connections, network protocols, and Google Accounts are fully verified. Try sending a mock transaction SMS to trigger background syncing automatically!

Self-Check Diagnostics Tool Included
